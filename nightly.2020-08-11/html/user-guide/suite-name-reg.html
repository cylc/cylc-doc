Suite Name Registration¶

Cylc commands identify suites via their names, which are relative path names under the suite run directory (~/cylc-run/ by default).

Suite names are validated by cylc.flow.unicode_rules.SuiteNameValidator.

class cylc.flow.unicode_rules.SuiteNameValidator¶

The rules for valid suite names:

  • must be between 1 and 254 characters long

  • can not start with: ., -

  • can only contain: alphanumeric, /, _, +, -, ., @

Suites can be grouped together under sub-directories. E.g.:

$ cylc print -t nwp
nwp
 |-oper
 | |-region1  Local Model Region1       /home/oliverh/cylc-run/nwp/oper/region1
 | `-region2  Local Model Region2       /home/oliverh/cylc-run/nwp/oper/region2
 `-test
   `-region1  Local Model TEST Region1  /home/oliverh/cylc-run/nwp/test/region1

Suite names can be pre-registered with the cylc register command, which creates the suite run directory structure and some service files underneath it. Otherwise, cylc run will do this at suite start up.
